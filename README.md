# ДЗ 15 — Браузерный поиск: история, подсказки, категории

## Цель:
закрепить и расширить навыки работы с реактивными формами и состоянием в Angular:
- Управление формой через FormGroup и FormControl
- valueChanges, реактивность, RxJS (опционально)
- Работа с localStorage для сохранения данных между сессиями
- Улучшение UX: история поиска, подсказки, категории
- Чистота кода, типизация, однонаправленные потоки данных

## Срок:
до следующего семинара — до **24.02.2026 в 23:00 мск**

---

### Основная часть — 9 баллов (по 3 балла)

#### A.1) История поиска (3 балла)
**1 балл — Отображение истории под полем поиска**  
- Под строкой поиска отображается список последних 5–10 запросов  
- Элементы истории отображаются в виде кликабельных кнопок или ссылок  
- История не содержит дубликатов (повторный запрос не добавляется дважды)

**1 балл — Восстановление поиска по клику**  
- При клике на элемент истории — значение подставляется в поле поиска  
- Форма корректно обновляется (через setValue/patchValue)  
- Фокус остаётся в поле (по желанию)

**1 балл — Сохранение и очистка истории**  
- История сохраняется в `localStorage` и восстанавливается при перезагрузке  
- Добавляется кнопка "Очистить историю" — удаляет историю из интерфейса и `localStorage`  
- При новом поиске (submit) запрос добавляется в начало истории

---

#### A.2) Подсказки (Autosuggest) (3 балла)

**1 балл — Отображение списка подсказок при вводе**  
- При вводе текста показываются подсказки из предопределённого массива (например: `['Angular', 'TypeScript', 'JavaScript', 'RxJS', 'NgRx']`)  
- Фильтрация — по вхождению подстроки (без учёта регистра)  
- Подсказки скрываются, если совпадений нет или поле пустое

**1 балл — Выбор подсказки**  
- При клике на подсказку — значение подставляется в поле поиска  
- Список подсказок скрывается  
- Подсказка работает корректно при любом порядке ввода (например, "script" → "JavaScript")

**1 балл — Управление видимостью и UX**  
- Подсказки скрываются при клике вне поля поиска  
- При повторном открытии — подсказки появляются снова, если есть ввод  
- Опционально: подсветка совпадающей части текста (через `innerHTML` и `<mark>`)

---

#### A.3) Категории поиска (3 балла)
**1 балл — Добавление выбора категории**  
- Добавить элемент выбора категории: `select` или группа кнопок-переключателей  
- Варианты: `Все`, `Новости`, `Видео`, `Картинки`, `Карты`  
- Категория управляется через отдельный `FormControl` внутри `FormGroup`

**1 балл — Привязка категории к форме и событию поиска**  
- При отправке формы — в консоль (или на экран) выводится объект: `{ query: '...', category: '...' }`  
- Категория корректно обновляется при изменении выбора

**1 балл — Сохранение категории между сессиями**  
- Выбранная категория сохраняется в `localStorage`  
- При загрузке страницы — значение подставляется в форму (через `patchValue`)  
- Значение по умолчанию: `Все`, если в `localStorage` нет данных

### Дополнительная часть — 1 балл (бонус)

#### B.1) Поиск с задержкой (debounceTime) — 1 балл
- Подписка на `valueChanges` поля поиска с использованием:  
  `pipe(debounceTime(300), distinctUntilChanged())`  
- После паузы в 300мс — в консоль выводится сообщение "Поиск: [значение]"  
- Добавляется индикатор "Поиск..." (текст или спиннер), который появляется во время задержки и исчезает после эмита  
- При быстром вводе — запросы не дублируются

> ⚠️ Не использовать debounce для отправки формы, только для логирования/подсказок

---

### Общие требования:
- Форма реализована через `ReactiveFormsModule`, `FormGroup`, `FormControl`
- Все данные — типизированы (использовать интерфейсы: `SearchQuery`, `Suggestion` и т.п.)
- Нет мутаций массивов напрямую — только через методы вроде `filter`, `map`, `concat`
- Код читаем, структурирован, без закомментированного "мусора"
- Стили — через классы, SCSS, минимальная вёрстка (flex/grid)

---

### Советы:
- Начните с истории — она проще всего и сразу виден результат  
- Подсказки — хорошее упражнение на `valueChanges` и управление DOM  
- Категории — потренируют работу с несколькими контролами в одной форме  
- `debounceTime` — не обязательно, но очень полезно для понимания RxJS

---

### Пример успешной реализации:
- Пользователь вводит "ang" → видит подсказку "Angular" → кликает → поле заполняется  
- Нажимает Enter → запрос "Angular" попадает в историю  
- Выбирает категорию "Видео" → ищет → перезагружает страницу → категория и история сохранены  
- Печатает быстро "ts" → "tsc" → "typescript" → сработает только один раз после паузы (если включён debounce)
