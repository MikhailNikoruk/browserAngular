# Практика 18-19 — Браузерный поиск: сервисы, DI, localStorage и тема

## Цель
Потренироваться проектировать и использовать **сервисы**:
- Разделять ответственность между UI и логикой
- Подключать сервисы через DI
- Хранить данные между сессиями через `localStorage` (через отдельный сервис)
- Сделать историю запросов через отдельный сервис
- Начать использовать RxJS в “простом” виде (опционально)
- Реализовать сервис темы и синхронизацию с `localStorage`

# Основная часть — 8 баллов

## A.1) LocalStorageService (2 балла)
**Задача:** сделать универсальный сервис-обёртку над `localStorage`.

**1 балл — Базовое API и типизация**
- Создать `LocalStorageService`
- Добавить методы чтения/записи/удаления по ключу
- Сервис умеет работать с объектами (сериализация/десериализация)
- Обработка ошибок: если JSON битый — не падать, возвращать значение по умолчанию

**1 балл — Применение в проекте**
- Использовать `LocalStorageService` минимум в 2 местах проекта (например, история и тема)
- **Подсказка:** такой сервис обычно делается “общим” (shared/core), но решение где хранить — за вами

---

## A.2) SearchHistoryService (2 балла)
**Задача:** историю запросов вынести из компонента в сервис и хранить между сессиями.

**1 балл — Логика истории в сервисе**
- Создать `SearchHistoryService`
- Сервис предоставляет:
  - получить список истории (последние N запросов)
  - добавить запрос в историю
  - очистить историю
- История:
  - не содержит дубликатов (повторный запрос переносится наверх)
  - ограничена по размеру (например, 10)

**1 балл — Интеграция с UI**
- Компонент(ы) не содержат бизнес-логики истории, только вызывают методы сервиса
- История отображается под формой и кликабельна (клик подставляет значение в форму)
- Есть кнопка “Очистить историю”

---

## A.3) ThemeService + синхронизация с localStorage (2 балла)
**Задача:** реализовать переключение темы через отдельный сервис.

**1 балл — Сервис темы**
- Создать `ThemeService`
- Сервис хранит текущую тему (например: `light | dark`)
- При старте приложения тема читается из `localStorage` (через `LocalStorageService`)
- При смене темы значение сохраняется в `localStorage`

**1 балл — Применение темы в UI**
- Добавить переключатель темы (кнопка/тумблер)
- Тема реально влияет на стили минимум 3 элементов:
  - фон страницы
  - карточки результатов
  - кнопки (обычная/активная)
- Реализация через CSS-классы на корневом контейнере (например, `theme-light` / `theme-dark`)

---

## A.4) SearchParamsService: единый источник параметров поиска (2 балла)
**Задача:** вынести параметры поиска из компонентов в отдельный сервис, чтобы управлять ими централизованно.

**1 балл — Сервис параметров и типы**
- Создать `SearchParamsService`
- Ввести тип `SearchParams` (минимум: `query`, `category`)
- Сервис умеет:
  - хранить текущие параметры
  - обновлять параметры частично (например, только `query` или только `category`)
  - сбрасывать параметры к значениям по умолчанию

**1 балл — Интеграция с формой и историей**
- При сабмите формы параметры обновляются через `SearchParamsService`, а не напрямую в компоненте
- При клике на элемент истории запрос подставляется в форму и **обновляет** параметры через сервис
- UI корректно отображает актуальные параметры после любых действий (submit, выбор истории, reset)

# Дополнительная часть — 2 балла (бонус)

## B.1) RxJS: реактивная история или подсказки (2 балла)
Выберите **один** вариант:

### Вариант 1 — История через RxJS (2 балла)
- В `SearchHistoryService` хранить историю как поток (например, observable)
- Компонент подписывается “правильно” (через async pipe или аккуратно управляя подпиской)
- При добавлении/очистке история в UI обновляется автоматически

### Вариант 2 — Подсказки через `valueChanges` (2 балла)
- Подписаться на `valueChanges` поля query
- Использовать `debounceTime(300)` и `distinctUntilChanged()`
- Показывать подсказки из предопределённого массива (как в прошлом ДЗ), но:
  - логика подсказок вынесена в сервис (например, `SuggestionsService`)
  - UI только отображает и отправляет события

- Всё типизировано (интерфейсы для query, темы, результата)
